Problem Definition

The goal of the project is to design and implement algorithms that delineate objects in video images and analyze their shapes. This is useful for motion detection in interactive applications. The problem is to create a program that can distinguish between different gestures using motion detection and image processing techniques learned in class. Here, we make the assumption that the only object in the viewing field is the object at hand; behind the object is a black wall that does not interfere with the calculations. This project is difficult because there are so many variables to either detect or ignore that it makes the algorithms extremely hard to implement due to interference from noise.

Method and Implementation

The gestures we’ve chosen to include in computer vision system are detected by creating a bounding box surrounding the centroid of the pixels that are within a color range of skin color. Using frame-to-frame differencing we are able to compare the previous frames and the previous size of the bounding box in time. By comparing the changing size of the bounding box to a threshold the application will judge whether a detected gesture, or change in the bounding box of the object, is one of the three defined gestures.

Using this method the application can detect three gestures: waving back and forth with one hand, making a fist, and moving towards or away from the camera.

Experiments

We ran tests for each of the gestures, and the fist and depth tests went well. For the waving gesture, there were a lot of false negatives which made qualitatively testing the waving gesture difficult. We performed 20 tests, with a probability of the waving being correct 30%, 50% for fist detection, and 80% for depth.

Discussion

Our method is strong in accurately detecting gestures if there is minimal background noise, unfortunately, the algorithm becomes a lot less accurate given the addition of new objects. In our experiments, we expected to be able to detect with our three supported gestures with a high probability of success, if there is minimal background interference.

If we had time, our program could be improved by labeling something which is moving as an object, rather than by labeling a mass of skin pixels as an object, because this would greatly reduce background interference due to the background being stable. We also could have implemented a database which the algorithm would reference in order to compare current gestures to.

Conclusions

Based on our findings, we conclude that it in a background noise free environment, implementing a gesture detection algorithm would be very easy, but given the reality of testing situations, it is unlikely that a perfect gesture detection algorithm but it is possible to write a system with a high probability of success.
